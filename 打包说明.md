# 基带测试数据统计工具 - 打包说明

## 打包方式

本项目提供了两种打包方式:

### 方式一: 文件夹模式（推荐）

**优点**: 启动速度快，文件结构清晰
**缺点**: 需要分发整个文件夹

**使用方法**:
1. 双击运行 `build.bat`
2. 等待打包完成
3. 打包结果在 `dist/基带测试数据统计工具/` 文件夹中
4. 运行 `dist/基带测试数据统计工具/基带测试数据统计工具.exe`

### 方式二: 单文件模式

**优点**: 只有一个exe文件，方便分发
**缺点**: 首次启动较慢（需要解压临时文件）

**使用方法**:
1. 双击运行 `build_onefile.bat`
2. 等待打包完成
3. 打包结果是 `dist/基带测试数据统计工具.exe`
4. 直接运行该exe文件

## 打包配置说明

### build.spec 文件

这是PyInstaller的配置文件，包含了:
- 入口文件: `main.py`
- 资源文件: `src/resources/styles.qss`
- 隐藏导入: PySide6和openpyxl相关模块
- 输出配置: 无控制台窗口的GUI应用

### 关键配置项

- `console=False`: 不显示控制台窗口
- `upx=True`: 使用UPX压缩（如果可用）
- `datas`: 包含QSS样式文件
- `hiddenimports`: 确保所有必需的模块被打包

## 打包后测试

打包完成后，建议进行以下测试:

1. 运行exe文件，检查界面是否正常显示
2. 测试文件上传功能
3. 测试充电测试数据处理
4. 测试续航测试数据处理
5. 检查输出文件是否正常生成

## 常见问题

### 1. 打包失败

- 确保虚拟环境已激活
- 确保所有依赖已安装: `pip list`
- 检查Python版本是否兼容

### 2. 运行时缺少模块

- 在 `build.spec` 的 `hiddenimports` 中添加缺失的模块
- 重新运行打包脚本

### 3. 资源文件找不到

- 检查 `datas` 配置是否正确
- 确保资源文件路径正确

### 4. exe文件过大

- 可以移除不必要的依赖
- 使用虚拟环境避免打包系统包
- 考虑使用UPX压缩

## 分发建议

### 文件夹模式分发

1. 将整个 `dist/基带测试数据统计工具/` 文件夹打包成zip
2. 用户解压后运行exe文件
3. 可以创建快捷方式到桌面

### 单文件模式分发

1. 直接分发 `dist/基带测试数据统计工具.exe`
2. 用户双击即可运行
3. 首次运行可能需要等待几秒
